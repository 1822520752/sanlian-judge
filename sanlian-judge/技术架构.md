## 4. LLM集成

### 4.1 DeepSeek API调用

```
模型: deepseek-chat
端点: https://api.deepseek.com/v1/chat/completions

请求格式:
  POST https://api.deepseek.com/v1/chat/completions
  Authorization: Bearer {DEEPSEEK_API_KEY}
  Content-Type: application/json

  {
    "model": "deepseek-chat",
    "messages": [
      {
        "role": "system",
        "content": "你是sanlian-judge..."
      },
      {
        "role": "user",
        "content": "请分析以下B站用户的赛博人格..."
      }
    ],
    "temperature": 0.9,
    "max_tokens": 2000,
    "response_format": { "type": "json_object" }
  }

成本估算:
  - 输入: ~500 token (系统prompt + 用户数据)
  - 输出: ~1000 token (JSON报告)
  - 单次: ~1500 token ≈ ¥0.0015
  - 1000次测试 ≈ ¥1.5

备选方案:
  - 本地模型: Ollama + Qwen2.5-7B (免费，但质量低)
  - 其他API: 通义千问（有免费额度）
```

### 5.2 前端状态管理

```javascript
// 极简状态管理，不需要框架
const state = {
  currentPage: 'home',    // 'home' | 'loading' | 'report'
  uid: '',
  profile: null,          // B站用户数据
  report: null,           // AI生成的报告
  ranking: null,          // 排行榜数据
};

// 页面切换
function showPage(page) {
  ['home', 'loading', 'report'].forEach(p => {
    document.getElementById(`page-${p}`).style.display = 'none';
  });
  document.getElementById(`page-${page}`).style.display = 'block';
  state.currentPage = page;
}
```

## 6. 部署方案

### 6.1 Vercel部署

```
部署流程:
  1. 将项目推送到GitHub
  2. 在Vercel导入项目
  3. 设置环境变量:
     - DEEPSEEK_API_KEY
     - (可选) VERCEL_KV_REST_API_URL
     - (可选) VERCEL_KV_REST_API_TOKEN
  4. Vercel自动检测Python项目，配置Serverless Functions
  5. 自动部署 → 获得 xxx.vercel.app 域名

文件配置:
  vercel.json:
  {
    "functions": {
      "api/**/*.py": { "runtime": "python3.11" }
    },
    "rewrites": [
      { "source": "/api/(.*)", "destination": "/api/$1" }
    ]
  }

requirements.txt:
  bilibili-api-python==17.4.2
  requests
```

## 7. 缓存策略

```
缓存规则:
  - 相同UID的报告结果缓存24小时
  - 存储方式: 前端localStorage + 后端可选
  - 触发条件: 用户重新测试同一UID时显示缓存结果
  - 过期后重新生成

缓存键: `report:{uid}`
缓存值: { profile, report, timestamp }

节省成本:
  - 假设10%用户重复测试 → 节省10% API调用
  - 1000次测试中节省100次 → 节省约¥0.15
```

## 8. 安全与合规

```
数据安全:
  - 不存储用户个人隐私数据
  - B站数据仅做临时使用，不持久化（排行榜除外）
  - LLM调用不发送敏感信息

合规:
  - 仅使用B站公开API
  - 报告标注"AI生成，仅供娱乐"
  - 不存储用户的登录凭据
  - 用户可以删除自己的排行榜记录

防滥用:
  - API调用频率限制 (可Vercel Edge Config实现)
  - 每个IP每分钟最多5次请求
  - 超长UID自动拦截
```
