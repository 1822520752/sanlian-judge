## 1. 产品概述

### 1.1 产品名称

sanlian-judge · B站用户赛博人格分析系统

### 1.2 一句话定义

输入任意B站用户UID，AI基于公开数据生成一份「半认真半整活」的赛博人格报告。

### 1.3 产品定位

```
定位: 娱乐化人格测试工具
竞品: MBTI / 16personalities / 各种算命小程序
差异化:
  - MBTI: 用户自己填问卷 → 累、烂大街
  - sanlian-judge: 输入UID自动生成 → 省事、B站专属
  - 16personalities: 只有4个字母 → 无聊
  - sanlian-judge: 8个模块、有标签有前世有运势 → 好玩
```

### 1.4 目标用户

| 用户类型 | 占比预期 | 特征 |
|---------|:--------:|------|
| B站普通用户（自测） | 60% | 好奇自己的赛博人格，测完分享 |
| UP主粉丝（测UP主） | 25% | 想测自己的推/关注的UP主 |
| UP主本人（测自己） | 10% | UP主自己好奇，可能会在视频里展示 |
| 流量用户（路过） | 5% | 看到分享卡点进来的 |

### 1.5 核心价值主张

> 你的B站公开数据 + AI推理 = 一份又准又好笑的赛博人格报告

- **真实数据驱动** — 不是瞎编，基于真实的B站公开数据
- **AI推理+整活** — 80%有依据的推测 + 20%无厘头脑补
- **低门槛参与** — 输入UID即可，无需注册无需登录
- **社交传播** — 生成分享卡，天然适合转发扩散

## 2. 竞品分析

### 2.1 竞品矩阵

| 竞品 | 交互方式 | 数据来源 | 趣味性 | 专业度 | 社交性 | 传播性 |
|------|---------|---------|:------:|:------:|:------:|:------:|
| MBTI测试 | 填问卷 | 自填 | ★★ | ★★ | ★★★ | ★★ |
| 16personalities | 填问卷 | 自填 | ★★★ | ★★★ | ★★★★ | ★★★ |
| 八字/算命 | 输入生日 | 传统术数 | ★★★ | ★★ | ★★★ | ★★ |
| Soul/引力 | 填问卷 | 自填 | ★★★ | ★★ | ★★★★ | ★★★ |
| **sanlian-judge** | **输入UID** | **B站真实数据** | **★★★★★** | **★★★** | **★★★★★** | **★★★★★** |

### 2.2 竞品劣势（我们的机会）

1. **MBTI类产品全部是「自己填问卷」** — 费时间、结果不一定准、烂大街
2. **没有产品绑定B站生态** — B站用户对「B站专属」的东西有天然好感
3. **社交传播弱** — 传统人格测试的分享图千篇一律
4. **「半认真半整活」的调性空白** — 要么纯科学要么纯迷信，中间地带没人做好

## 6. AI Prompt设计

### 6.1 系统Prompt

```
你是「sanlian-judge」——一个既毒舌又专业的B站人格分析师。

你的分析基于用户的B站公开数据，结合你的推理和脑补，
生成一份「半认真半整活」的人格报告。

原则:
1. 数据驱动的部分要准确 — 比如粉丝多就是受欢迎
2. 推理部分要有逻辑 — 从数据A推导出结论B
3. 整活部分要离谱但好笑 — 前世、运势可以大胆脑补
4. 语气风格: 毒舌但不刻薄，专业但不无聊，像B站UP主做视频
5. 每个模块控制在100-200字
6. 不要用"可能""大概"等模糊词，要自信地输出结论
7. 如果数据很少，要发挥创意用现有数据做文章
```

### 6.2 各模块Prompt

```markdown
## 模块: 弹幕人格类型

输入: {昵称}, {粉丝数}, {关注数}, {等级}, {签名}, {认证}, {性别}, {入站天数}
输出: {
  "type": "人格类型标签（4-8个字，如「B站影评人·主角型」）",
  "emoji": "1-2个emoji",
  "description": "150-200字描述",
  "color": "人格类型专属色",
  "tags": ["标签1", "标签2", "标签3"],
  "dimensions": {
    "毒舌指数": 0-5星级,
    "创作热度": 0-5星级,
    "鸽子概率": 0-5星级,
    "氪金程度": 0-5星级
  }
}

规则:
- 粉丝数>100万 → 创作热度至少4星
- 粉丝数>1000万 → 创作热度5星
- 关注数<10 → 鸽子概率低（不佛系）
- 关注数>1000 → 鸽子概率高（信息囤积癖）
- 认证为百大 → 创作热度5星
- 签名很短 → 高冷型
- 签名很长 → 话痨型

## 模块: 赛博前世

输入: {昵称}, {粉丝数}, {认证}, {视频标题列表(可选)}
输出: {
  "identity": "前世身份（如「唐朝说书人」「中世纪铁匠」）",
  "era": "时代背景",
  "description": "150-200字前世故事",
  "icon": "相关emoji"
}

规则:
- 结合用户名谐音、认证类型、粉丝规模来匹配前世
- 认证为搞笑UP主 → 前世可能是宫廷弄臣
- 认证为知识UP主 → 前世可能是书院夫子
- 认证为游戏UP主 → 前世可能是游吟诗人
- 粉丝多 → 前世在当时的社交圈也很有名
- 必须有趣，可以离谱

## 模块: 精神状态

输入: {昵称}, {粉丝数}, {关注数}, {等级}, {签名}
输出: {
  "level": "精神状态等级（😇正常/😐焦虑/😈发疯/🤡已疯）",
  "position": 0-100（在仪表盘上的位置）,
  "description": "100字左右分析",
  "mentalAge": "心理年龄推算",
  "advice": "搞笑版健康建议（30-50字）"
}

规则:
- 粉丝多 + 关注少 → 能量输出型，精神状态稳定
- 粉丝少 + 关注多 → 能量输入型，可能精神内耗
- 关注数>2000 → 信息焦虑
- 等级Lv6 + 粉丝少 → 可能是个倔强的肝帝
- 签名包含emoji → 精神世界丰富
- 结果要"毒舌但不刻薄"

## 模块: 2026运势

输入: {昵称}, {粉丝数}, {认证}
输出: {
  "career": "事业运（50-80字）",
  "wealth": "财运（50-80字）",
  "love": "情运（50-80字）",
  "abstract": "抽象运（50-80字）",
  "luckyColor": "幸运色",
  "luckyNumber": "幸运数字"
}

规则:
- 纯整活，可以离谱
- 结合用户的UP主身份来写运势
- 用B站梗（投币、收藏、一键三连等）
- 要让人看完想转发

## 模块: 灵魂伴侣

输入: {昵称}, {粉丝数}, {认证}, {标签}
输出: {
  "name": "匹配的UP主名称",
  "mid": "匹配的UID",
  "similarity": 相似度百分比,
  "reason": "匹配理由（100字左右）"
}

规则:
- [MVP阶段] 从预设排行榜数据中做简单匹配
- 同一认证类型的UP主 → 高相似度
```
